# Розділ 3: План Розробки (Roadmap)

Цей план розбиває процес розробки MVP на логічні фази з орієнтовними термінами та ключовими цілями для кожної з них.

### Фаза 0: Налаштування Проєкту (Тиждень 0)

*   **Налаштування середовища:** Створення репозиторію, ініціалізація проєкту Next.js 16 з TypeScript, встановлення основних бібліотек (shadcn/ui, Zustand, Supabase).
*   **Ініціалізація Supabase:** Створення проєкту в Supabase, розгортання схеми бази даних за допомогою SQL-скриптів, налаштування автентифікації.
*   **Ключі для інтеграцій:** Отримання тестових ключів API для «Нова Пошта» та email-сервісу. Зберігання їх у змінних середовища.
*   **Архітектура коду:** Створення базової структури папок відповідно до методології Feature-Sliced Design.
*   **Розгортання:** Підключення репозиторію до Vercel для налаштування безперервного розгортання (CI/CD).

**Ключовий результат:** Готове до розробки середовище, розгорнуте в хмарі.

### Фаза 1: Розробка Ядра Функціоналу (Тижні 1-2)

*   **Автентифікація та організації:** Реалізація реєстрації, входу в систему. Автоматичне створення організації при реєстрації.
*   **Основний макет:** Створення головного лейауту з навігацією (бічна панель, хедер), адаптованого для мобільних та десктопних пристроїв.
*   **Модуль "Клієнти":**
    *   Сторінка зі списком клієнтів, отриманих з Supabase.
    *   Форма для створення/редагування клієнта.
    *   Реалізація м'якого видалення.
    *   Перевірка роботи RLS-політик.
*   **Модуль "Угоди" (базова версія):**
    *   Сторінка зі списком угод.
    *   Форма створення угоди з вибором клієнта та можливістю додати товарні позиції.
    *   Зміна статусу угоди.

**Ключовий результат:** Користувач може зареєструватися, додати своїх клієнтів та створити перші угоди. Основний кістяк CRM готовий.

### Фаза 2: Інтеграції та Офлайн-режим (Тижні 3-4)

*   **Інтеграція з «Нова Пошта»:**
    *   Кнопка "Створити ТТН" на сторінці угоди.
    *   Виклик API Нової Пошти та збереження отриманого номера ТТН.
    *   Посилання на відстеження посилки.
*   **Email-сповіщення:**
    *   Реалізація відправки шаблонного email-повідомлення при зміні статусу замовлення на "Відправлено".
*   **Реалізація офлайн-режиму (PWA):**
    *   Налаштування Service Worker для кешування ресурсів.
    *   Впровадження IndexedDB для зберігання даних.
    *   Реалізація черги синхронізації для операцій, виконаних офлайн (пріоритет — створення нових записів).
    *   Тестування роботи в офлайн-режимі через інструменти розробника.
*   **Імітація підписок:**
    *   Додавання логіки перевірки тарифного плану.
    *   Реалізація одного жорсткого ліміту (напр., кількість контактів) для демонстрації роботи обмежень.

**Ключовий результат:** MVP функціонально завершений. Працює ключова інтеграція та реалізована головна конкурентна перевага — офлайн-режим.

### Фаза 3: Тестування та Доопрацювання (Тиждень 5)

*   **Комплексне тестування:** Перевірка всіх користувацьких сценаріїв на різних пристроях та браузерах.
*   **Виправлення багів:** Усунення критичних помилок, виявлених під час тестування.
*   **Тестування безпеки:** Ручна перевірка надійності RLS-політик (спроба отримати доступ до чужих даних).
*   **Перевірка відповідності (Compliance):** Переконатися, що всі вимоги (напр., посилання на Політику конфіденційності) виконані.
*   **Підготовка до запуску:** Фіналізація налаштувань продакшн-середовища, перевірка змінних середовища.

**Ключовий результат:** Стабільний, протестований та готовий до запуску для перших користувачів MVP.

### Фаза 4: Пост-MVP Розвиток (Подальший план)

*   **Збір фідбеку:** Аналіз відгуків перших користувачів та пріоритезація подальших доопрацювань.
*   **Розширення модулів:** Послідовна реалізація функціоналу, що не увійшов до MVP (складський облік, розширена аналітика, автоматизація).
*   **Масштабування інфраструктури:** Планування та реалізація міграції на власні сервери при зростанні навантаження.
*   **Маркетинговий сайт:** Створення повноцінного лендингу для залучення нових клієнтів.
