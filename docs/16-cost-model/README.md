# Розділ 16: Модель Витрат

Розуміння структури витрат на різних етапах розвитку проєкту є ключовим для планування масштабування та ціноутворення. Наша стратегія полягає в мінімізації витрат на старті з поступовим їх зростанням, що корелює з ростом доходів.

### Етап 1: MVP на безкоштовних тарифах

На цьому етапі ми максимально використовуємо безкоштовні плани хмарних сервісів, щоб звести операційні витрати практично до нуля.

*   **Supabase (Free Plan):** $0/міс. Включає 500 МБ бази даних, 1 ГБ сховища файлів, 50,000 активних користувачів на місяць. Цього більш ніж достатньо для запуску та тестування.
*   **Vercel (Hobby Plan):** $0/міс. Включає 100 ГБ трафіку та достатньо ресурсів для запуску MVP.
*   **Інтеграції:** API «Нова Пошта» та email-сервіси (напр., SendGrid) мають безкоштовні плани, яких вистачить для початкового етапу.
*   **Домен:** Єдина потенційна стаття витрат — реєстрація доменного імені (~$10-15/рік).

**Загальні витрати на етапі MVP: ~$1/міс.**

### Етап 2: Раннє зростання (оновлення керованих сервісів)

Коли використання продукту перевищить ліміти безкоштовних планів, ми перейдемо на платні тарифи, що є простішим та швидшим рішенням, ніж одразу мігрувати на власні сервери.

*   **Supabase (Pro Plan):** ~$25/міс. Надає 8 ГБ бази даних, 100 ГБ сховища, знімає обмеження на "засинання" проєкту та надає кращу підтримку.
*   **Vercel (Pro Plan):** ~$20/міс за одного розробника. Може знадобитися при зростанні трафіку, але, ймовірно, ми швидше переїдемо на власний VPS, ніж будемо платити значні суми за перевищення лімітів Vercel.
*   **Інші витрати:** Можлива оплата за розширені ліміти email-розсилок або SMS-шлюзів (якщо вони будуть інтегровані).

**Орієнтовні витрати на етапі раннього зростання: ~$25-50/міс.**

### Етап 3: Масштабування (перехід на власні/орендовані сервери)

Коли щомісячні платежі за керовані сервіси стануть невигідними порівняно з вартістю оренди власного сервера, буде здійснено міграцію (див. Розділ 15).

*   **Оренда VPS:** ~$40-80/міс за віртуальний сервер, здатний розмістити всі компоненти системи (додаток, БД, Redis, черги).
*   **Сховище файлів (S3-сумісне):** Використання сервісів на кшталт Backblaze B2 або AWS S3 для зберігання файлів буде коштувати ~$5-10/міс за значні обсяги.
*   **Резервні копії, моніторинг:** Додаткові, але незначні витрати на надійність інфраструктури.

**Орієнтовні витрати на етапі масштабування: ~$50-150/міс.**

### Загальна стратегія та порівняння

| Етап | Інфраструктура | Орієнтовні витрати/міс | Примітки |
|---|---|---|---|
| **MVP** | Supabase (Free), Vercel (Hobby) | **~$0-1** | Повністю в межах безкоштовних квот. |
| **Раннє зростання** | Supabase (Pro), Vercel (Pro) | **~$25-50** | Підтримка помірного зростання без значних інвестицій. |
| **Масштабування** | Власний/орендований VPS, Docker | **~$50-150+** | Більший контроль, фіксовані витрати, вигідніше при великій кількості користувачів. |
| **Enterprise** | Кластер серверів, керовані БД | **$500+** | Для тисяч клієнтів; витрати повністю покриваються доходами. |

### Співвідношення доходів та витрат
Ключова бізнес-мета — забезпечити, щоб доходи від підписок випереджали ріст витрат на інфраструктуру. Навіть на етапі раннього зростання, дохід від кількох клієнтів (напр., 5 клієнтів по ~ $20/міс = $100) має з запасом покривати витрати (~$45/міс). Важливо **відстежувати вартість інфраструктури на одного активного користувача**, щоб контролювати юніт-економіку.

### Витрати на White-label
Для клієнтів на тарифі Enterprise (White-label), які можуть потребувати окремого, ізольованого середовища, модель витрат буде індивідуальною. Витрати на їхню інфраструктуру (напр., окремий VPS за ~$50/міс) будуть повністю закладені в ціну преміум-тарифу (напр., $200-500/міс), що забезпечить високу маржинальність.

### Резервний план (Contingency)
*   **Аномалії витрат:** Необхідно постійно моніторити витрати на сторонні сервіси (особливо ті, що тарифікуються за використання, як-от SMS), щоб уникнути несподіваних рахунків.
*   **Валютні ризики:** Оскільки витрати на сервери, ймовірно, будуть в USD, а доходи — в UAH, необхідно враховувати валютні коливання при ціноутворенні.

**Висновок:** Стратегія полягає в тому, щоб витрати на інфраструктуру росли повільніше, ніж доходи від підписок. Перехід на власні сервери планується тоді, коли це стане економічно вигідніше, ніж оплата керованих сервісів. Наприклад, коли рахунки від Supabase+Vercel перевищать $100/міс, оренда власного VPS за $50-80/міс стане очевидним кроком. Це забезпечує фінансову життєздатність проєкту на кожному етапі його життєвого циклу.
